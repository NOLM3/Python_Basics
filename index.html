Introduction

Python is a high-level, general-purpose programming language known for its simplicity and readability. It was created by Guido van Rossum and first released in 1991. Python is designed to be easy to understand and use, making it an ideal choice for beginners and experienced developers alike.

One of Python’s core philosophies is its emphasis on code readability. This is reflected in its use of indentation to define code blocks, instead of curly braces like many other languages. This design helps make Python code more concise and intuitive to read, which contributes to its popularity.

Python is versatile and widely used in a variety of fields such as web development, data science, automation, artificial intelligence, and scientific computing. It has a large community, which means there is an extensive library of tools and frameworks to help with almost any kind of task. Some well-known Python frameworks include Django for web development, Pandas for data analysis, and TensorFlow for machine learning.

Some key features of Python include:

  • Simple and Readable Syntax: Python's syntax is designed to be straightforward and readable, making it a great choice for beginners.
  • Dynamic Typing: Python uses dynamic typing, meaning variables do not need to be explicitly declared with a type. The type is inferred based on the assigned value.
  • Interpreted Language: Python code is executed by an interpreter, which means it runs line by line and doesn't require a compilation step. This allows for easier debugging and rapid development.
  • Cross-Platform Compatibility: Python is available on multiple platforms, including Windows, macOS, and Linux, making it highly portable.
  • Extensive Libraries: Python has a rich ecosystem of libraries and frameworks, from web development to data science and machine learning, which greatly enhance its versatility.

In the following sections, key concepts such as data types, variables, control flow, and functions will be introduced to provide a solid foundation for Python programming.

Strings and the print() Function

In Python, **strings** are sequences of characters enclosed in single (' ') or double (" ") quotes. Python allows you to use both single and double quotes for creating strings, which makes it easier to include quotes inside the string without the need for escape characters.

Examples of strings:

  • 'Hello World'
  • "Python is awesome"

You can also use double quotes inside strings defined by single quotes, and vice versa:

  • 'This is a "string" example'
  • "She said 'Hello' and left"

For longer strings or strings that need line breaks, you can use **triple quotes** (single or double). This allows you to create multi-line strings without needing to use the newline escape character (`\n`).

  • '''This is a string spanning multiple lines'''
  • """This also works and can include multiple line breaks."""

String Concatenation: You can combine multiple strings using the concatenation operator +. Additionally, strings can be repeated using the repetition operator *.

  • greeting = 'Hello ' + 'World' # Result: 'Hello World'
  • repeat = 'Python ' * 3 # Result: 'Python Python Python '

print() Function: The print() function is one of the most commonly used functions in Python. It is used to display information to the console or standard output. You can print text, variables, results of operations, and more.

The print() function can accept multiple arguments separated by commas, and Python will insert a space between them when printing.

  • print('Hello', 'World') # Result: 'Hello World'
  • print('The result of the operation is', 10 + 5) # Result: 'The result of the operation is 15'

Additionally, you can control how the text is printed, such as the separator between arguments, by using the sep parameter:

  • print('Python', 'is', 'awesome', sep='-') # Result: 'Python-is-awesome'

The print() function also allows you to change the character that ends the line. By default, it adds a newline character at the end, but if you don't want that, you can modify it using the end parameter:

  • print('Hello', end=' ') # Result: 'Hello ' without a newline
  • print('World', end='!\n') # Result: 'World!' with an exclamation mark at the end

F-strings: Starting from Python 3.6, f-strings allow you to embed expressions inside string literals in a very readable and efficient way. Using the syntax f'...' , you can directly insert values within curly braces {}.

  • name = 'John'
  • print(f'Hello, {name}!') # Result: 'Hello, John!'

F-strings also allow you to perform calculations or invoke functions inside the curly braces:

  • age = 25
  • print(f'In 5 years, you will be {age + 5} years old.') # Result: 'In 5 years, you will be 30 years old.'

Escaping Characters: If you need to include special characters inside strings, such as quotes or line breaks, you can use the escape character (\) to prevent them from being interpreted as string delimiters.

  • print('He said: \"Python is awesome\"')
  • print('First line\nSecond line')
Reserved Words

In Python, **reserved words** (also called keywords) are words that have a specific meaning in the language's syntax. These words are predefined by Python and cannot be used as identifiers (e.g., variable names, function names, class names). Each reserved word serves a particular purpose in the language, and they help define the structure of the Python code.

Reserved words are part of the core Python syntax and are used to represent language constructs such as control flow (e.g., if, else), data types (e.g., int, str), and other important language features like loops, exception handling, and function definitions.

For example: The word if is used to define conditional statements, while def is used to define functions.

Reserved words cannot be used for naming variables, functions, or other identifiers in your Python programs. Attempting to use a reserved word as an identifier will result in a syntax error. For instance, trying to assign a value to a variable named def would raise an error:

  • def = 5 # This will raise a SyntaxError

There are currently 35 reserved words in Python 3.x. Some of the most common reserved words include:

  • False - Represents a boolean value, false.
  • None - Represents a null value or "no value here".
  • True - Represents a boolean value, true.
  • and - A logical operator used to combine conditional statements.
  • as - Used to create an alias while importing modules or packages.
  • class - Used to define a class.
  • def - Used to define a function.
  • if - Used to start a conditional block.
  • import - Used to import a module.
  • try - Used to start a block of code for exception handling.

These reserved words are case-sensitive, which means that True and true are different. In Python, True refers to the boolean value, while true would be treated as an identifier and could lead to errors if used incorrectly.

For a complete list of reserved words in Python, you can use the keyword module, which provides a programmatic way to access the full list:

  • import keyword
  • print(keyword.kwlist)

Understanding reserved words is essential when writing Python programs to avoid naming conflicts and ensure that your code is syntactically correct.

Variables and Rules

In Python, variables are used to store data that can be accessed and manipulated throughout your program. A variable is essentially a name that points to a value, and it is created by assigning a value to a name using the assignment operator =.

Python is a dynamically typed language, meaning you don’t need to declare the data type of a variable when you create it. Python will automatically infer the type based on the value assigned to it. For example:

  • name = "Alice"
  • age = 25
  • height = 5.8

In Python, there are certain rules and conventions you must follow when naming variables:

  • Variable names must start with a letter (a-z, A-Z) or an underscore (_). They cannot start with a number.
  • After the first character, variable names can include letters, numbers, and underscores.
  • Variable names are case-sensitive. For example, Age and age are considered two different variables.
  • Variable names cannot be the same as Python reserved words (keywords).

Examples of Correct Syntax:

  • my_var = 10
  • my_var = "Hello"

As seen above, my_var initially holds an integer value, but later in the program, it is assigned a string. This flexibility is one of the advantages of Python's dynamic typing system.

Numeric Expressions

In Python, numeric expressions allow you to perform mathematical operations on numbers. Python supports several basic arithmetic operations, including addition, subtraction, multiplication, and division. Additionally, Python also includes advanced operations like exponentiation and modulus operations.

Here are the most common arithmetic operators in Python:

  • Addition (+): Adds two numbers.
  • Subtraction (-): Subtracts one number from another.
  • Multiplication (*): Multiplies two numbers.
  • Division (/): Divides one number by another, always returning a float.
  • Exponentiation (**): Raises a number to the power of another number.
  • Modulo (%): Returns the remainder of a division.
  • Floor Division (//): Divides and returns the largest integer less than or equal to the result.

Examples of numeric expressions in Python:

result = 10 + 5 * 2

This expression first multiplies 5 * 2 (which equals 10), and then adds 10 to that result, giving a final value of 20.

result = 9 // 4

The floor division operator (//) divides 9 by 4, giving 2 (ignoring the remainder).

result = 10 % 3

The modulus operator (%) returns the remainder of the division of 10 by 3, which is 1.

Python follows the standard order of operations (PEMDAS) for numeric expressions:

  • Parentheses first
  • Exponentiation
  • Multiplication and Division (from left to right)
  • Addition and Subtraction (from left to right)

For example:

result = (3 + 5) * 2

This expression evaluates as (3 + 5) * 2, giving a result of 16 because parentheses have the highest precedence.

Order of Evaluation

In Python, the order of evaluation determines the sequence in which operators are applied in an expression. Python follows the standard mathematical rules for evaluating expressions, often referred to as the operator precedence.

Python evaluates expressions according to the following hierarchy, which is also known as PEMDAS:

  • Parentheses: Operations inside parentheses are evaluated first.
  • Exponentiation: The exponentiation operator (**) is evaluated after parentheses.
  • Multiplication, Division, Floor Division, and Modulo: These operations are performed next, from left to right.
  • Addition and Subtraction: Finally, addition and subtraction are evaluated, also from left to right.

This hierarchy ensures that expressions are evaluated consistently and according to mathematical standards. For example, multiplication and division are performed before addition and subtraction unless overridden by parentheses.

Examples:

result = 3 + 5 * 2

In this expression, multiplication (*) is evaluated before addition (+). The result is 13 because the multiplication 5 * 2 = 10, and then 3 + 10 = 13.

result = (3 + 5) * 2

In this case, the parentheses force the addition to be evaluated first. So, (3 + 5) = 8, and then 8 * 2 = 16.

If you are unsure about how an expression will be evaluated, it is a good practice to use parentheses to make your intent clear, especially in complex expressions.

String to Int/Float Conversion

In Python, it is common to need to convert strings that represent numbers into actual numeric data types (integers or floats) so you can perform mathematical operations on them. Python provides two built-in functions for this purpose: int() and float().

The int() function converts a string that represents an integer into an actual integer. If the string does not represent a valid integer, Python will raise a ValueError.

  • int(): Converts a string representing an integer to an integer type.
  • float(): Converts a string representing a floating-point number to a float type.

For example:

num = int("10")

This converts the string "10" to the integer 10. Similarly:

decimal = float("10.5")

converts the string "10.5" to the float 10.5.

It's important to remember that when you are trying to convert a string to a number, the string must contain a valid numeric value. If the string contains any characters that are not part of a number, Python will raise an error:

int("abc")

If you attempt to convert the string "abc", it will result in a ValueError because "abc" cannot be interpreted as a number.

You can also use int() and float() with numbers written in a different format. For example, the int() function can convert a string with leading or trailing spaces:

num = int(" 25 ")

This will correctly convert the string " 25 " into the integer 25.

When working with floating-point numbers, you can also handle strings that use scientific notation:

number = float("1e3")

This converts the string "1e3" to the float 1000.0, which is the result of the expression 1 * 10^3.

Remember that these conversions only work with valid numeric strings. If you need to handle user input or external data that may not always be a valid number, it's a good idea to use try-except blocks to handle potential conversion errors gracefully:

try:
    num = int("abc")
except ValueError:
    print("Invalid input! Not a number.")

In this case, if the conversion fails, the except block will catch the error and print a message instead of crashing the program.

Indentation/Deindentation

Indentation is a crucial aspect of Python syntax. Unlike many programming languages that use braces or keywords to define blocks of code, Python uses indentation to indicate blocks of code. Proper indentation is required for the code to run correctly.

Python uses a standard indentation of 4 spaces (not tabs) for each block level. When writing Python code, each block of code that belongs to a control structure (such as if, for, while, etc.) must be indented consistently. Improper indentation will cause a IndentationError.

For example:

if x > 5:
    print("x is greater than 5")

In this example, the statement print("x is greater than 5") is part of the if block and must be indented by 4 spaces. If you remove the indentation or add too many spaces, Python will raise an error:

if x > 5:
print("x is greater than 5")

Deindenting is the process of reducing the level of indentation to exit a block. For example, after the block inside the if statement is finished, you return to the previous indentation level to continue with the rest of the code:

if x > 5:
    print("x is greater than 5")
print("This is outside the if statement.")

In this example, the second print() statement is not indented, so it will execute regardless of the if condition.

It's also important to note that Python does not allow mixing tabs and spaces for indentation. This can lead to confusing errors. Always configure your text editor or IDE to insert spaces when you press the tab key. Most editors, including Visual Studio Code, will handle this automatically if you set it to use spaces for indentation.

Here's another example of indentation used with loops:

for i in range(3):
    print("Iteration:", i)

In this loop, the print() function is indented, indicating that it is part of the for loop. After the loop finishes, the next lines of code should be at the previous indentation level.

Lastly, proper deindenting ensures that your program structure remains correct. For instance:

def my_function():
    print("This is inside the function")
print("This is outside the function")

In this case, the second print() is outside the function and not indented. Indentation should always reflect the logical structure of your program.

Control Flow

Control flow refers to the order in which individual statements, instructions, or function calls are executed or evaluated in a program. Python uses control flow statements such as if, elif, else, and loops (for, while) to allow developers to create decision-making and repetitive processes in their programs.

The most common control flow statements are:

  • if: Used to test a condition and execute a block of code if the condition is true.
  • elif: Used to test multiple conditions. If the if condition is false, it checks the next condition.
  • else: Executes a block of code if none of the if or elif conditions are true.
  • for: Used to iterate over a sequence (like a list, tuple, or range).
  • while: Used to execute a block of code as long as a specified condition is true.

If Statements

The if statement tests a condition and executes the code block if the condition is true.

if x > 5:
    print("x is greater than 5")

If the condition x > 5 is true, Python will execute the code inside the block, which is the print() statement. If the condition is false, Python will skip the block.

Elif and Else

The elif statement allows you to test multiple conditions. If the first condition in the if block is false, Python moves on to the elif conditions. If none of the conditions are true, the else block will be executed.

if x > 10:
    print("x is greater than 10")
elif x == 5:
    print("x is 5")
else:
    print("x is less than or equal to 10 but not 5")

In this example, if x > 10 is true, the first block will run. If not, Python will check if x == 5. If that condition is true, it will print "x is 5". Otherwise, it will print "x is less than or equal to 10 but not 5".

For Loops

A for loop is used to iterate over a sequence of elements, such as a list, string, or range of numbers. It repeatedly executes a block of code for each item in the sequence.

for i in range(3):
    print(i)

This loop will print the numbers 0, 1, and 2. The range(3) function generates the sequence of numbers from 0 to 2.

While Loops

A while loop continues to execute a block of code as long as the specified condition is true. If the condition becomes false, the loop stops.

while x < 5:
    print(x)
    x += 1

This while loop will print the value of x starting from its initial value and increment it by 1 each time, as long as x < 5. Once x reaches 5, the loop will stop.

Break and Continue Statements

In addition to if, for, and while loops, Python has break and continue statements that control loop execution:

  • break: Exits the loop entirely, regardless of whether the condition is still true.
  • continue: Skips the current iteration of the loop and moves to the next one.

For example:

for i in range(5):
    if i == 3:
        break
    print(i)

This loop will print 0, 1, and 2, then exit the loop when i == 3.

for i in range(5):
    if i == 3:
        continue
    print(i)

This loop will print 0, 1, 2, and 4, skipping the number 3 due to the continue statement.

Nested If Statements

You can also nest if statements inside other if statements. This allows for more complex decision-making structures:

if x > 5:
    if x == 10:
        print("x is exactly 10")
    else:
        print("x is greater than 5 but not 10")

In this example, if x > 5 is true, Python will check if x == 10. If that is true, it will print "x is exactly 10", otherwise, it will print "x is greater than 5 but not 10".
